# F1TI · 测测你的 F1TI

> **项目介绍文档(评审版)**
> 一款以上海国际赛车场为蓝本、面向抖音互动空间的纯离线 H5 体感赛车小游戏

---

## 一、项目概览

| 字段 | 内容 |
|---|---|
| **项目名称** | F1TI · 测测你的 F1TI |
| **形态** | 纯离线 HTML5 单文件应用(Three.js + TypeScript)|
| **包体** | 单文件 `index.html` ≈ 8 MB,ZIP 后 ≈ 5 MB,完全符合 8 MB 上限 |
| **目标平台** | 抖音互动空间 / 移动端浏览器 / 桌面浏览器 |

**一句话介绍:**

> 我们打造了一款面向 **F1 与上海赛车文化爱好者**,在 **抖音碎片化使用场景** 下,通过 **多模态操控 + AI 双语音陪跑 + 真实赛道几何**,带来 **"在上海国际赛车场跑一圈"沉浸式体验** 的小游戏;并在赛后基于玩家驾驶数据,自动匹配 **14 位上海站现役 F1 车手** 中最相似的一位,生成可分享的 MBTI 风格"赛车人格"卡片。

## 二、为什么做(Why)

### 2.1 受众与情境

| 维度 | 描述 |
|---|---|
| 年龄段 | 18 – 35 岁 |
| 身份 | F1 / 赛车赛事观众、抖音互动空间用户、上海本地年轻人 |
| 兴趣偏好 | 体育竞技、模拟驾驶、人格测试、社交分享 |
| 触达情境 | 抖音 feed 互动空间入口、F1 大奖赛话题页、通勤与赛事间隙的碎片时段 |

**核心洞察:**
F1 是国内"声量高、认知低"的体育项目。大众听说过周冠宇、汉密尔顿,但很难说清不同车手的驾驶风格差异。这一"认知缝隙"正是我们切入的核心机会。

### 2.2 用户痛点与设计初衷

我们围绕三类痛点设计游戏体验:

**① 赛事氛围难以延续**
F1 中国大奖赛仅在每年 4 月举办一次,赛后观众缺乏延续赛事热情的轻量入口。

**② 模拟器门槛过高**
F1 25、iRacing 等正版模拟器装机包 80 GB 以上,需要方向盘外设,移动端完全不可触达,普通观众无法上手。

**③ 现有小游戏与"上海"和"F1"无关**
抖音互动空间内现有赛车小游戏多为顶视角跑酷,操控方式单一(左右两键),既无真实赛道,也无车手风格化呈现。

### 2.3 真实场景观察

> **观察 1**:2025 年 F1 中国大奖赛期间,B 站 #F1 上海大奖赛 话题播放量超 1.2 亿,但评论区高频问题是"周冠宇是什么风格""不同车手的驾驶差异在哪里"。说明大众有兴趣,缺工具。

> **观察 2**:抖音互动空间已上线的赛车类小游戏,操控方式普遍仅支持"左右两键",缺乏移动端原生传感器(陀螺仪)的差异化体验,留存率与转化率有限。

> **观察 3**:正版 F1 模拟器与抖音轻量小游戏之间存在显著体验空白。前者重度沉浸但门槛高,后者轻量易上手但与 F1 无关 —— 中间没有产品。

### 2.4 我们想填补的空白

| 类别 | 代表方案 | 局限 |
|---|---|---|
| 重度模拟器 | F1 25 / iRacing | 80 GB+ 安装包、需手柄、学习曲线陡峭 |
| 现有 H5 小游戏 | 通用跑酷 / 卡丁车 | 顶视图、操控单一、与 F1 无关、无社交钩子 |
| **F1TI(本项目)** | 离线单文件 | **真实上海赛道 / 五种操控 / 双语音陪跑 / F1TI 人格结果卡** |

## 三、做了什么(What)

### 3.1 核心功能 ① · 多模态操控系统

**功能名称:** 五种操控方式 + 自动降级容错

**设计意图:**
抖音用户的设备能力差异极大(从 iPhone 15 到入门安卓、从 MacBook 到 Windows 笔记本)。单一操控会直接劝退一半用户。我们让游戏自动检测设备传感器能力,提供最适合的方式,并在不可用时优雅降级。

**功能矩阵:**

| 操控方式 | 适用设备 | 核心逻辑 |
|---|---|---|
| 键盘 | 桌面浏览器 | WASD / 方向键 / 空格 / Shift(DRS)|
| 触屏 | 普通智能手机 | 左右半屏分区按住即转向 |
| **体感(陀螺仪)** | 移动端横屏 | 左右倾 = 转向;**前后倾 = 油门 / 刹车** |
| **鼠标虚拟摇杆** | 无传感器笔记本(如 MacBook) | 鼠标偏屏幕中心 = 摇杆位移 |
| **iPhone 当外接手柄** | 开发模式 / 同 WiFi | 局域网 WebSocket,iPhone 当 IMU 传感器 |

**关键流程:**

```
菜单选择操作方式
    ↓
检测设备能力(matchMedia / DeviceOrientationEvent)
    ↓
请求权限(iOS 严格在用户手势内调用 requestPermission)
    ↓
传感器有数据 → 启用所选模式
    ↓
传感器无数据 → 自动 fallback(键盘 / 鼠标摇杆)
```

**技术亮点:**
- 体感信号经死区 + S 曲线 + EMA 滤波处理,横屏自动适配 `screen.orientation`
- iOS Safari 严格在 button click 用户手势同步调用 `requestPermission`,绕过沙盒静默拒绝
- 任何模式失败都不会让游戏卡死,均有兜底路径

### 3.2 核心功能 ② · AI 解说员 + 驾驶教练双语音引擎

**功能名称:** 比赛解说 + 实时教练,语音陪跑系统

**设计意图:**
单人比赛容易感到孤独,新手无法判断弯道节奏。我们用两套独立的语音系统解决这两个问题,玩家可在菜单中三选一(关闭 / 解说 / 辅助驾驶)。

#### 解说模式 — 打造"看真比赛"的代入感

| 维度 | 描述 |
|---|---|
| 事件检测 | 24 个驾驶事件(起跑爆发 / 第一弯 / 急刹 / 出界 / 重撞 / 超车 / 半程 / 冲刺 / 冠军 / 季军等)|
| 音频资产 | 31 段预录中文解说 mp3,VBR 80 kbps,总计 1.7 MB |
| 防刷屏机制 | 三层冷却(per-event + 全局 2.2 秒间隔 + 优先级抢占)|
| 结尾差异化 | 根据最终名次和撞墙次数,自动播放"完美夺冠""勉强完赛""登上领奖台"等不同结尾 |

#### 教练模式 — 让新手也能跑得动

| 维度 | 描述 |
|---|---|
| 实现方式 | 浏览器内置 SpeechSynthesis API,**零额外音频资源** |
| 核心算法 | 每帧采样赛道前方 80 米的 8 个曲线点,累计角度差判断弯道方向与严重度 |
| 触发逻辑 | 角度 > 24° → 转向提示;角度 > 49° + 速度 > 100 km/h → 刹车提示;长直道持续 1.8 s → 加速提示 |

**核心价值:**
- 解说模式提升单人比赛沉浸感
- 教练模式显著降低新手过弯门槛,提高完赛率与留存率

### 3.3 核心功能 ③ · 赛车人格 MBTI 卡片(社交核心)

**功能名称:** 赛后自动匹配上海站 F1 车手人格,生成可分享卡片

**设计意图:**
玩家不只是为了赢,更是为了知道"我像哪位 F1 车手"。这一机制把私人比赛数据转化为可在朋友圈、抖音传播的社交标签,提供持续重玩动机。

**算法核心:**

```
比赛实时数据
    ↓
12 维驾驶风格映射
(pace, consistency, clean, cornering, braking, racingLine,
 attack, defense, risk, comeback, pressure, management)
    ↓
对 14 位上海站现役 F1 车手画像进行加权欧氏距离匹配
    ↓
最高匹配度 = 你的赛车人格
    ↓
动态生成 4 条"为何你是这个类型"原因
    ↓
渲染成 MBTI 风格卡片(纯 HTML / CSS,留照片槽位)
```

**输出结构(JSON,前端直接消费):**

```json
{
  "你的赛车人格": {
    "类型名称": "单圈天赋型",
    "匹配车手": "Charles Leclerc",
    "匹配度": 98,
    "一句话总结": "你的单圈速度和弯道天赋非常突出..."
  },
  "为何你是这个类型": [
    "你在整体速度上表现突出,能快速建立比赛节奏。",
    "你在弯道中展现出鲜明的节奏感...",
    "你的刹车点判断果断...",
    "你能在单圈里榨干极限,把弯道当成自己的舞台。"
  ],
  "核心标签": ["高速", "弯道", "极限", "高光"],
  "相似人格Top3": [...]
}
```

**核心价值:**
- 14 位车手画像直接对应 2025 年上海站现役 grid,具有时效性与话题性
- 卡片留出车手照片槽位,可后续接入官方授权图片
- 动态原因生成基于玩家个人数据,而非固定模板,保证唯一性

## 六、技术架构亮点

为评审快速理解本项目工程深度,简述关键技术决策:

- **完全离线:** 单文件 HTML 内联所有 JS / CSS / 图片(base64)/ 音频,生产构建无任何网络请求
- **真实赛道几何:** 65 个手工放置的赛道控制点,Cubic Bezier 曲线插值,T11–T13 区域用数学纯净的圆弧重构,消除 Catmull-Rom 自交伪影
- **物理模型:** 自研轻量物理(转向 / 油门 / 刹车 / 智能修正 / 出界判定),与 AI 共用同一个赛道投影函数
- **InstancedMesh 优化:** 460 栋天际线 + 240 个云朵粒子合并到 ≤ 5 个 draw call,移动端 60 FPS 稳定
- **状态机驱动:** MENU → SCAN → PICK_TEAM → COUNTDOWN → RACE → FINISH → RESULT 全流程状态机化,易扩展易测试

当前确认正确的 Red Bull 轮胎方案标记为 `redbull-github-v1`，只允许应用于 `RB19_REDBULL.opt.glb`。其他车型保持独立的 `pending` 策略，禁止复用 Red Bull 的轮组识别。

|---|---|

| Red Bull GLB SHA-1 | `66c78ca97b11d3cbaf20f2bf9c7eec7a2614d3ae` |

修改车辆、车库或模型加载流程后，必须执行 `npm run verify:redbull-wheels`。校验失败时不得继续用其他车型参数覆盖 Red Bull 逻辑。

F1TI 不仅是一款赛车小游戏,也是一次通过驾驶表现测出玩家 F1TI 人格的互动体验。我们希望:

- **任何人** 都能在 8 MB 的轻量包内,跑一圈中国大奖赛主场
- **任何设备** 都能找到适合自己的操控方式
- **任何水平的玩家** 都能在陪跑语音中获得提升
- **每一局比赛** 都能转化为可分享的人格故事

这是我们送给抖音用户、F1 观众和上海这座城市的小礼物。
